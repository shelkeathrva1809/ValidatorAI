# ValidatorAI

## Features

- **AI-Powered Validation** -- Submit a startup idea and receive a comprehensive investor-grade analysis report generated by Google Gemini (or OpenRouter as a free alternative).
- **Market Sizing** -- TAM, SAM, and SOM estimates with CAGR growth rates, market drivers, and segment breakdowns.
- **Competitor Intelligence** -- Real competitors discovered via Google Search grounding (Gemini), categorized as Direct or Indirect, with strengths and weaknesses.
- **Technical Feasibility** -- Complexity scoring, suggested tech stack, development timeline, and key engineering challenges.
- **Financial Projections** -- Funding estimates, break-even timeline, revenue streams, and cost structure.
- **Risk Assessment** -- Critical risks identified and ranked, with a strategic roadmap of next steps.
- **GO / CAUTION / NO-GO Verdict** -- A clear recommendation with rationale, designed to mirror how a VC evaluates pitches.
- **Report Dashboard** -- Browse, view, and delete saved reports. Data persists across sessions via SQLite.
- **Source Citations** -- When using Gemini with Google Search grounding, reports include links to real sources backing the analysis.

## Getting Started

### Prerequisites

- **Node.js** (v18 or later recommended)
- **npm**
- An API key from one of:
  - [Google AI Studio](https://aistudio.google.com/apikey) (Gemini)
  - [OpenRouter](https://openrouter.ai/keys) (free tier available)

### Installation

```bash
# Clone the repository
git clone <your-repo-url>
cd validatorai

# Install dependencies
npm install
```

### Environment Setup

Create a `.env.local` file in the project root (or edit the existing one):

```env
# Option A: Google Gemini (recommended -- enables Google Search grounding)
GEMINI_API_KEY=your_gemini_api_key_here

# Option B: OpenRouter (free alternative)
OPENROUTER_API_KEY=your_openrouter_api_key_here
OPENROUTER_MODEL=google/gemini-2.0-flash-exp:free
```

> If both keys are present, OpenRouter takes priority. Remove `OPENROUTER_API_KEY` to use Gemini.

### Running the App

**Frontend only** (reports won't persist to database):

```bash
npm run dev
```

**Frontend + Backend** (full experience with SQLite persistence):

```bash
npm run dev:all
```

Or run them separately in two terminals:

```bash
# Terminal 1 -- Backend API server
npm run start:server

# Terminal 2 -- Vite dev server
npm run dev
```

The frontend runs on `http://localhost:3000` and proxies `/api` requests to the backend on `http://localhost:8787`.

### Production Build

```bash
npm run build
npm run preview
```

### How It Works

1. User describes their startup idea on the **New Analysis** form.
2. The idea is sent to the **AI service** (`services/gemini.ts`), which calls either Google Gemini or OpenRouter.
3. The AI returns a structured JSON report following a strict schema covering market data, competitors, financials, technical feasibility, and risks.
4. When using Gemini, the AI calls Google Search during generation to ground its analysis in real data and returns source citations.
5. The report is saved to **SQLite** (via the Express backend) and mirrored to **localStorage** for instant UI access.
6. The user can browse all reports on the **Dashboard**, view full details, or delete them.

### Data Flow

```
User Input ──> gemini.ts (AI Service) ──> Gemini API / OpenRouter API
                                                  │
                                          Structured JSON Report
                                                  │
                      ┌───────────────────────────┤
                      │                           │
              storage.ts ──> POST /api/reports    localStorage
                                  │
                           SQLite Database
```

## API Reference

The backend server exposes a REST API on port `8787`.

| Method | Endpoint | Description |
|--------|----------|-------------|
| `GET` | `/api/health` | Health check. Returns `{ "ok": true }` |
| `GET` | `/api/reports` | List all reports, ordered by most recent |
| `GET` | `/api/reports/:id` | Get a single report by ID |
| `POST` | `/api/reports` | Create or update a report (upsert). Send full Report JSON in body |
| `DELETE` | `/api/reports/:id` | Delete a report. Returns `204 No Content` |

### Database Schema

```sql
CREATE TABLE reports (
  id              TEXT PRIMARY KEY,
  createdAt       INTEGER NOT NULL,
  title           TEXT,
  ideaDescription TEXT,
  json            TEXT NOT NULL        -- Full Report object serialized as JSON
);
```

## Configuration

### Environment Variables

| Variable | Required | Description |
|----------|----------|-------------|
| `GEMINI_API_KEY` | One of these | Google Gemini API key |
| `OPENROUTER_API_KEY` | is required | OpenRouter API key |
| `OPENROUTER_MODEL` | No | OpenRouter model slug. Default: `google/gemini-2.0-flash-exp:free` |

### Recommended OpenRouter Models

| Model | Strengths |
|-------|-----------|
| `google/gemini-2.0-flash-exp:free` | Fast, strong structured JSON output |
| `google/gemma-3-27b:free` | Deeper reasoning, slower |

> **Note:** OpenRouter does not support Gemini's Google Search grounding tool. Reports will still be high quality, but source citations will not be available.

### Ports

| Service | Default Port | Override |
|---------|-------------|----------|
| Vite dev server | 3000 | `vite.config.ts` |
| Express backend | 8787 | `PORT` env variable |

## Report Schema

Each analysis report follows this structure:

```typescript
interface Report {
  id: string;                    // UUID v4
  title: string;                 // AI-generated title
  ideaDescription: string;       // Original user input
  createdAt: number;             // Unix timestamp (ms)
  summary: string;               // 2-3 sentence executive summary
  decision: "GO" | "CAUTION" | "NO-GO";
  decisionRationale: string;

  market: {
    tam: string;                 // e.g. "$45B"
    sam: string;
    som: string;
    growthRate: string;          // e.g. "12.5%"
    overview: string;            // Detailed market analysis paragraph
    marketDrivers: string[];
    segments: string[];
  };

  competitors: Array<{
    name: string;
    type: "Direct" | "Indirect";
    description: string;
    strengths: string[];
    weaknesses: string[];
  }>;

  businessModel: string;

  technical: {
    complexity: "Low" | "Medium" | "High" | "Very High";
    suggestedStack: string[];
    developmentTimeline: string;
    keyChallenges: string[];
  };

  financials: {
    initialFundingEstimate: string;
    breakEvenTimeline: string;
    revenueStreams: string[];
    costStructure: string[];
  };

  risks: string[];
  nextSteps: string[];
  sources: Array<{ title: string; uri: string }>;
}
```
